# Onstell v0.1.0-beta.4 Release Checklist

This checklist prepares the next development beta. It should not be published as a user-ready KVM release.

## Release Positioning

Beta.4 is for smoke testing the desktop shell after Milestone 2 and Milestone 3 planning, simulation, and safety-model work.

Clearly state:

- Onstell is still not a working KVM.
- Routing is modeled and simulated only.
- Input capture is not active by default and has only a development-only guard.
- The loopback follower is in-process and fake.
- Pointer edge detection, keyboard normalization, emergency shortcut planning, and pairing trust records are pure local models only.
- Pairing/session transport is designed but not implemented.
- Clipboard sync remains design-only.
- No real keyboard, pointer, clipboard, LAN control, or file-transfer traffic is sent.

Keep [v0.1.0-beta.3](https://github.com/real-CAK3D/Onstell/releases/tag/v0.1.0-beta.3) available as the first clean desktop-shell smoke-test release.

## Included Since beta.3

- Routing and permission readiness panel.
- Pure TypeScript routing state machine.
- Release Input action in the widget and tray.
- Read-only input capture spike plan and fail-closed development guard.
- Local in-process loopback follower simulator.
- Trusted LAN pairing/session transport design.
- Fixed-controller mode guard.
- Platform permission readiness model.
- Pure pointer edge detection model for simulated coordinates.
- Pure keyboard normalization model that excludes typed text and blocks release/secure shortcuts.
- Dev-only emergency release shortcut guard; no shortcut is registered.
- Local-only fake pairing trust store prototype.
- Wiki pages for the Milestone 2 safety model.

## Failed Draft Cleanup

If a beta.4 release workflow creates a bad draft:

1. Do not publish it.
2. Download any useful logs from the failed workflow.
3. Delete the failed draft release from GitHub Releases.
4. Delete any accidentally created duplicate `v0.1.0-beta.4` tag only if it points to the wrong commit.
5. Re-run the release workflow from the intended commit.

Do not delete beta.3 while preparing beta.4.
