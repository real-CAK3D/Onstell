# Contributing

Onstell is at the design and prototype stage.

## Ground Rules

- Keep user-facing claims accurate.
- Do not label Device Desktop mode as a real extended monitor.
- Keep Virtual Monitor mode separate from pointer/keyboard sharing.
- Prefer small, reviewable changes.
- Document platform-specific limitations instead of hiding them.

## Development Flow

1. Create a branch.
2. Make focused changes.
3. Run relevant checks.
4. Open a pull request.

## UI Direction

The widget should feel like a compact glass control puck:

- Purple-first identity.
- Cyan, rose, and amber supporting accents.
- Readable at reduced opacity.
- Hover-revealed controls.
- Clear active-device state.
- No false "connected" state before real pairing exists.
